# 用户认证系统需求文档
# trustforge parse 将从此文件生成需求切片

## 用户注册

用户通过提供用户名和密码进行注册。
- 用户名：3-20 个字符，只能包含字母和数字
- 密码：至少 8 个字符
- 注册成功后用户状态为"未激活"
- 需要发送激活邮件

## 用户激活

用户通过点击邮件中的激活链接来激活账号。
- 激活后状态变为"已激活"
- 激活链接 24 小时有效
- 激活链接只能使用一次

## 用户登录

已激活的用户可以通过用户名和密码登录。
- 登录成功后返回 JWT token
- 登录失败次数超过 5 次，账号被锁定
- 锁定时间 30 分钟

## 管理员解锁

管理员可以手动解锁被锁定的账号。
- 解锁后状态恢复为"已激活"
- 登录失败次数清零

## 密码重置

已登录用户可以修改密码。
- 需要提供旧密码验证
- 新密码不能与旧密码相同
- 修改成功后需要重新登录
